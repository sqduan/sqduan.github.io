范例——软件注册机程序

引言

编写目的

本文为软件注册机程序的需求说明书，针对软件注册机用户的需求进行详细阐述。本文档面向软件注册机程序的开发者和使用者。

项目背景

项目涉及单位
  • 委托单位——XXX软件使用者
  • 开发单位——中国科学技术大学
  • 主管部门——网络安全部
系统关系

本软件与其他系统无明确关系

版本管理

术语 定义 缩略词原文
CPU 计算机系统的运算和控制核心 central processing unit
CPU序列号 CPU独一无二的ID号

参考资料

任务概述

目标

开发一xxx软件的软件注册机，能够对xxx软件进行加密，防止恶意盗取软件等行为，从而实现软件保护的功能。软件应具有注册码生成以及软件注册等功能，具体如下：

  • 能够获取用户硬件信息，包括CPU序列号、键盘序列号及网卡地址等信息
  • 能够根据用户输入的有效信息生成对应注册码并对软件进行注册，对无效信息进行过滤屏蔽
  • 在一定限制范围内允许用户对软件进行试用
  • 在删除软件后能够清除相应的注册表文件

运行环境

  • 操作系统 Windows7以上
  • CPU Intel i3 以上
  • 512MB内存
  • 10GB硬盘
  • 百兆带宽或以上

条件与限制

数据描述

表态数据

无

动态数据

输入数据
名称 类型 功能 范围 精度
用户名 string 软件使用者，注册机可根据用户名生成对应注册码
输出数据
名称 类型 功能 范围 精度
注册码 string 注册码可以对xxx软件进行注册，从而开启软件正常功能

功能描述

注册码生成

根据用户CPU、磁盘、网卡等信息，生成注册码，并操作注册表对软件进行注册

获取CPU序列号

根据CPU类型获取相应CPUID

获取磁盘序列号

根据磁盘类型获取磁盘序列号

获取网卡地址

根据网卡类型及网络相关信息，获取网卡的地址

软件注册

为用户提供软件试用或软件注册等功能

软件试用

允许用户在一定次数范围内试用软件

软件注册

通过用户输入的名称对软件进行注册，从而允许用户完整地使用xxx软件

性能需求

精度

用户名必须为有效ASCII字符，且必须以字母或下划线开头，有效字符包括数字、字母、下划线。长度应限定在1-20个字符内。

时间特性要求

  • 软件开启时间50ms
  • 按键响应时间20ms
  • 新窗口打开时间50ms
  • 数据更新时间5ms
  • 注册表注册时间10ms

运行需求

硬件接口

支持X86 64 位的i3系列以上CPU

软件接口

运行于Windows 7及更高版本支持WIN64 API的操作系统

通信接口

TCP/IP

其他需求

安全性

限制注册次数，禁止用户暴力破解

可维护性

可移植性
